# 2026-05-13 — iter-10 draft (fallback if iter-9 fails)

If iter-9 (plo teardown `civac` → `ivac` + iter-8 kernel changes) still
hits the translation-fault-L3 at FAR=0xffffffffc0001890, the next move
is iter-10: **enable SCTLR.C=1 IMMEDIATELY after SCTLR.M=1, before any
kernel PT writes**. With NC_BLOCK_ATTRS already Cacheable (kept from
iter-8), the page-table writes flow through D-cache, the walker reads
through D-cache, and the two stay coherent without explicit
maintenance.

This is the closest cousin to Linux's `__enable_mmu` pattern that
fits the current Phoenix code shape.

## Why iter-10 might succeed where iter-9 didn't

iter-9 keeps PT writes uncached (C=0) and tries to bridge the gap with
`dc ivac` before turning C=1. The risk is that A72's L2 has lines
that ivac doesn't reach (e.g., due to a hardware quirk), or that
speculation refills cache lines between ivac and walker access.

iter-10 eliminates the bridge: writes go IN to cache, walker reads
THROUGH the same cache. Whatever speculation does, the cache state
matches RAM because cache is the authoritative source.

The risk for iter-10 is that early atomic exclusives (LDAXR/STLXR for
nCpusStarted at line ~538) may behave differently with C=1. They
should work — Normal Cacheable mappings are the standard target for
exclusives — but it's worth watching for an EC=0x21 (PC alignment)
or EC=0x25 (data abort on exclusive monitor) signature in the fault
dump.

## Fallback after iter-10

If iter-10 also fails, the next escalation is iter-11: **move all PT
construction BEFORE SCTLR.M=1** (the true Linux `__cpu_setup`
pattern). This requires:

* Moving the stack pointer init to use raw PA (adrp works pre-MMU).
* Skipping or relocating the LDAXR/STLXR for nCpusStarted (which
  requires MMU + cacheable mapping for exclusives to be defined).
* Setting up `_other_core_trap` before secondaries can possibly
  reach the kernel image (though current SMP smoke parks them in
  armstub WFE, so this is moot for now).

iter-11 is a much bigger change. Save for a separate session.
